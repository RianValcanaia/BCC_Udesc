Guia de estudo

🧠 Mapa de Estudos — Estruturas de Dados (Teórico)

🔹 Árvore Binária de Busca (BST / ABB)

  • Definição e propriedades básicas
  • Condição de ordenação (esquerda < raiz < direita)
  • O que acontece quando desbalanceia (ex: vira lista ligada)
  • Complexidade (melhor caso e pior caso)

🔹 Árvore de Huffman

  • Definição e objetivo (compressão de dados)
  • Como a árvore é construída (com base nas frequências dos símbolos)
  • Funcionamento da codificação (0 para esquerda, 1 para direita)
  • Vantagem de ser ótima para compressão sem perda
  • Aplicações reais (ZIP, JPEG, MP3 etc.)
  • Diferença entre codificação de prefixo e codificação arbitrária

🔹 Árvore AVL

  • Definição e conceito de fator de balanceamento
  • Como identificar desbalanceamentos (fator = ±2)
  • Tipos de rotação: simples e dupla (esq, dir, esq-dir, dir-esq)
  • Vantagens em relação à BST comum
  • Altura garantida

🔹 Árvore Rubro-Negra

  • Definição (propriedades de cor, regras da estrutura)
  • Conceito de balanceamento por cores
  • Comparação com AVL (menos rigorosa, mais eficiente para inserções)
  • Casos onde é mais vantajosa

🔹 Árvore 2-3

  • Definição (nós com 1 ou 2 chaves, até 3 filhos)
  • Inserção e como divide/une nós
  • Como ela se mantém balanceada naturalmente
  • Ligação com árvore Rubro-Negra (equivalência estrutural)

🔹 Árvore B

  • Definição (árvore balanceada de ordem M)
  • Por que é usada em disco (alta ramificação, poucos acessos)
  • Inserção/remoção com divisão e fusão de nós
  • Diferença entre árvore B e 2-3

🔹 Árvore B+

  • Diferença entre B e B+ (apenas folhas armazenam chaves)
  • Vantagem para busca sequencial e intervalar
  • Como funciona o encadeamento das folhas
  • Uso em bancos de dados

🔹 Trie

  • Definição (estrutura baseada em prefixos de strings)
  • Como busca é feita letra por letra
  • Aplicações típicas (autocomplete, dicionário)
  • Problemas de uso (uso de memória)
  • Otimizações possíveis (Trie compactada / Patricia)

🔹 Tabela Hash

  • Definição e conceito de função hash
  • Como ocorrem as colisões (encadeamento, sondagem)
  • Diferença entre hash perfeito e com colisão
  • Por que não é usada para dados ordenados
  • Casos de uso (mapas, caches, dicionários)
