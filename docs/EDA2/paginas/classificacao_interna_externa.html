📦 Classificação em Memória Principal e Secundária

💡 Introdução

Relembrando o desafio do arquivo sequencial: não é possível acessar posições aleatórias diretamente.

🔹 Exemplo: para ler o 10º registro, é necessário ler os 9 anteriores.

⚠️ Agora imagine um arquivo com 10⁹ registros… ler o último pode ser extremamente custoso.

Programas podem gastar muitos recursos computacionais e tempo com:

  • Classificação de registros
  • Ordenação
  • Pesquisas
  • Agrupamento de dados

    🔍 Neste resumo, usaremos o termo classificação como sinônimo de ordenação.

📑 Tipos de Classificação

A classificação dos registros de um arquivo pode ser de dois tipos:

  • 🔹 Classificação Interna: usa somente a memória principal. O arquivo inteiro cabe nela.
  • 🔹 Classificação Externa: usa memória secundária, pois o arquivo não cabe na RAM.

    ⚠️ Aqui, é essencial minimizar o número de operações de E/S (entrada e saída) com o disco.

💾 Classificação Interna

Utiliza exclusivamente a memória principal, mantendo todo o arquivo nela.

✅ Critério de eficiência: número de comparações entre chaves e registros.

🔸 O processo consiste em:

  • Ler m registros por vez.
  • Classificar esses registros.
  • Gravar em uma partição.
  • Todas as partições têm m registros, exceto a última, que pode ter menos.

🛠️ Técnicas de Classificação Interna

Organizadas por estratégia:

🔹 Seleção:

  • Bubble Sort
  • Selection Sort
  • Heap Sort

🔹 Dividir para conquistar:

  • Merge Sort
  • Quick Sort

🔹 Inserção:

  • Insertion Sort
  • Shell Sort

🔹 Linear:

  • Bucket Sort
  • Radix Sort

💽 Classificação Externa

A classificação externa pode ser feita:

  • 🔹 Sobre o próprio arquivo:

    ✅ Economiza espaço.

    ⚠️ Risco de inconsistência caso o processo termine de forma anormal.

  • 🔹 Usando armazenamento auxiliar (arquivo temporário):

    ✅ Mais seguro, mas usa mais espaço.

📌 A estratégia básica é:

  1. Dividir o arquivo em frações menores.
  1. Ordenar essas frações.
  1. Intercalar as partes em dois estágios:
    • Classificação
    • Intercalação

🧩 Estratégia da Classificação Externa

🔍 Definição:

  • Arquivo de entrada: não classificado.
  • Arquivo de saída: classificado.
  • Uma partição é uma sequência ordenada de n registros.

⚙️ Procedimento da Classificação

  1. Os registros são lidos do arquivo de entrada ou das partições.
  1. São classificados.
  1. Depois, são gravados no arquivo de saída ou em novas partições.

🧪 Métodos de Classificação Externa

  • 📌 Classificação Interna (usada como etapa auxiliar)
  • 📌 Seleção com Substituição
  • 📌 Seleção Natural

✅ Considera-se que a memória comporta até m registros do arquivo.

🔄 Seleção com Substituição

🎯 Objetivo: aproveitar uma classificação parcial do arquivo de entrada para gerar partições maiores e mais ordenadas.

📋 Pseudo-algoritmo

  1. Ler M registros do arquivo para a memória
  1. Selecionar no vetor em memória o registro r com menor chave
  1. Gravar o registro r na partição de saída
  1. Substituir no vetor em memória o registro r pelo próximo registro do arquivo de entrada
  1. Se a chave deste último for menor do que a chave recém-gravada, considerá-lo congelado e ignorá-lo no restante do processamento
  1. Caso existam em memória registros não congelados, voltar ao passo 2
  1. Caso contrário:
    • Fechar a partição de saída
    • Descongelar os registros congelados
    • Abrir nova partição de saída
    • Voltar ao passo 2

📏 Tamanho das Partições

✅ Em média, as partições geradas têm tamanho ≈ 2 × m,

sendo m o número de registros lidos do arquivo.

⚠️ Desvantagem

No final de uma partição, boa parte da memória pode estar ocupada com registros congelados, o que reduz a eficiência da próxima.

🌿 Seleção Natural

A estratégia da seleção natural é reservar um espaço de memória secundária (chamado de reservatório) para armazenar os registros congelados do processo de substituição.

A formação de uma partição se encerra quando:

  • O reservatório estiver cheio, ou
  • Os registros de entrada tiverem sido todos processados.

Para uma memória com capacidade de M registros e um reservatório com n registros (normalmente M = n), o comprimento médio das partições é:

📋 Pseudo-algoritmo

  1. Ler M registros do arquivo para a memória
  1. Selecionar no vetor em memória o registro r com menor chave ainda não gravado na partição de saída
  1. Gravar o registro r na partição de saída
  1. Caso o reservatório não esteja cheio, substituir no vetor em memória o registro r pelo próximo registro do arquivo de entrada
  1. Enquanto a chave deste último for menor do que a chave recém-gravada:
    • Gravá-lo no reservatório
    • Substituir no vetor em memória o registro r pelo próximo registro do arquivo de entrada
  1. Se ainda existir espaço livre no reservatório ou registros na memória ainda não gravados:
    • Voltar ao passo 2
  1. Caso contrário:
    • Fechar a partição de saída
    • Copiar os registros do reservatório para o vetor em memória
    • Esvaziar o reservatório
    • Abrir nova partição de saída
    • Voltar ao passo 2

Comparação dos Métodos

📌 A classificação interna gera as menores partições, o que implica em mais arquivos a intercalar.

📈 Já os métodos de seleção com substituição e seleção natural geram partições maiores, o que reduz o tempo total de processamento.

⚠️ A seleção natural tem como desvantagem o uso de mais operações de entrada e saída, já que o reservatório também fica na memória secundária (arquivo).

Intercalação das Partições Classificadas

🎯 O objetivo da intercalação é juntar um conjunto de partições classificadas em um único arquivo, contendo todos os registros de todas as partições originais.

📌 O arquivo final é classificado com o mesmo critério das partições que o originaram.

🔁 Considerando que o processo anterior gerou R partições, a intercalação será feita em várias etapas, onde:

  • Os registros são lidos de um conjunto de partições,
  • E gravados em um novo arquivo de saída.

🌲 Árvore Binária de Vencedores

Essa estrutura de árvore é usada para intercalar partições classificadas de forma eficiente.

Como funciona:

  • Os nós folha da árvore contêm as chaves no topo das pilhas das partições a serem intercaladas.
  • Cada nó interno armazena a menor chave entre seus dois filhos.
  • A raiz da árvore sempre representa o menor valor entre todas as partições.

Cada nó da árvore contém:

  • 🔹 Vencedor: valor da menor chave da subárvore iniciada por esse nó;
  • 📍 Endereço do vencedor: ponteiro para a partição que possui aquela chave;
  • ⬅️ Esquerda: ponteiro para o filho da esquerda;
  • ➡️ Direita: ponteiro para o filho da direita.

📋 Pseudo-algoritmo

  1. Colocar na memória o primeiro registro de cada partição (cada registro será um nó folha da árvore);
  1. Criar um nó pai para cada par de folhas, com o menor valor entre as duas;
  1. Repetir o passo 2 até formar a raiz da árvore;
  1. Retirar a raiz (menor chave) e gravá-la no arquivo classificado;
  1. Adicionar um novo valor no nó folha vazio correspondente;
  1. Voltar ao passo 2 enquanto ainda houver registros a serem processados.

📈 Complexidade Algorítmica

  • A construção inicial da árvore tem complexidade O(n), onde n é o número de partições;
  • A cada iteração (inserção no arquivo final), são feitas log₂(n) comparações;
  • O número total de iterações é igual ao número de registros no arquivo.

⚠️ Limitação prática:

Nem sempre é possível intercalar todas as partições de uma só vez, pois a árvore pode ultrapassar a capacidade de memória. Além disso, o sistema operacional pode limitar o número de arquivos abertos simultaneamente, afetando o processo de intercalação.

Estágio de intercalação

🔁 A intercalação exige uma sequência de iterações:

  • Os registros são lidos de um conjunto de partições;
  • E gravados em outras partições, até que reste apenas uma totalmente classificada.

⚙️ Estratégias de Distribuição e Intercalação

  • 🔄 Intercalação balanceada de N caminhos

    (N partições são intercaladas ao mesmo tempo, balanceando a entrada e saída).

  • 🎯 Intercalação ótima

    (Busca reduzir ao mínimo o número total de passos de leitura e gravação).

📏 Medida de Eficiência

  • A eficiência da intercalação é medida pelo número de passos;
  • Isso representa o número médio de vezes que um registro é lido ou gravado durante todo o processo.

🔄 Intercalação Balanceada de N Caminhos

  • Determina-se o número de arquivos F que o algoritmo vai manipular:
    • A primeira metade dos arquivos será usada para leitura (entrada);
    • A segunda metade, para escrita (saída).
  • 📊 Distribuir as partições o mais igualitariamente possível entre os arquivos de entrada.
  • 🔃 Intercalar duas partições e gravar o resultado em uma nova partição em um dos arquivos de saída.
  • 🕘 Ao final de cada rodada:
    • As partições gravadas nos arquivos de saída tornam-se as de entrada na rodada seguinte.
    • O processo termina quando uma única partição é gravada.

Exemplo de Rodadas:

  1. 20 partições com 100 registros cada
  1. 10 partições com 200 registros cada
  1. 5 partições com 400 registros cada
  1. 2 partições com 800 registros cada + 1 com 400 registros
  1. 1 partição com 1600 registros + 1 com 400 registros
  1. ✅ Resultado final: 1 partição com 2000 registros

Intercalação Ótima

  • 📁 Determina-se o número de arquivos F a usar:
    • F - 1 arquivos para entrada
    • 1 arquivo para saída
  • 🔄 Em cada rodada, F - 1 partições são intercaladas e o resultado é gravado em um único arquivo de saída.
  • 🧹 Após cada rodada:
    • As partições intercaladas são removidas;
    • A partição gerada é adicionada a algum arquivo de entrada.
  • ✅ O algoritmo termina quando o conjunto de entrada tiver apenas uma partição, ou seja, o arquivo final classificado.

🕵️‍♂️ Pesquisa em Memória Principal e Secundária

🔍 Pesquisa

A pesquisa é a ação de recuperar uma informação dentro de um conjunto de dados. Assim como a ordenação, a operação de pesquisa é de grande importância e amplamente utilizada em diversos tipos de software. O objetivo da pesquisa é encontrar uma ou mais ocorrências de registros com chaves iguais à chave de acesso. Ela pode resultar em sucesso ou falha.

Cada unidade de informação é mantida em uma estrutura complexa, contendo uma chave primária, além dos dados que compõem a informação. O conjunto de registros normalmente é armazenado em tipos abstratos de dados, como:

  • Estruturas lineares (listas e suas variações);
  • Estruturas hierárquicas (árvores binárias e variações).

As rotinas de pesquisa devem ser eficientes, ou seja, devem realizar o processo com o menor número de iterações possível. O número de iterações depende do algoritmo de pesquisa escolhido, que deve ser selecionado com base em fatores como:

  • Quantidade de registros envolvidos;
  • Frequência de operações de inserção e exclusão de registros.

🧑‍💻 Algoritmos de Pesquisa

Pesquisa em Memória Principal:

  • Pesquisa sequencial (linear) ou com sentinela;
  • Pesquisa binária;
  • Pesquisa por interpolação;
  • Pesquisa direta (hashing).

Pesquisa em Memória Secundária:

  • Árvore binária;
  • Árvore B, B+ e B*;
  • Árvore Trie e Patricia.

🔎 Pesquisa Sequencial ou Linear

Este é um método simples de pesquisa, utilizado quando os dados não estão ordenados pela chave de acesso.

Princípio:

  • A pesquisa começa pelo primeiro registro e avança sequencialmente (registro por registro), até encontrar o registro desejado.
  • Caso de sucesso: A chave pesquisada é encontrada;
  • Caso de falha: Todos os registros são visitados e a chave não é encontrada.

👨🏻‍💻 Algoritmo em C++ de pesquisa sequencial ou linear

int linearSearch(int chave, int vetor[], int tamVetor){
		for (int i = 0; i < tamVetor; i++){
				if (vetor[i] == chave) return i;
		}
		return -1; // indice inválido
}

🔎 Pesquisa Sequencial com Sentinela

O algoritmo de pesquisa sequencial pode ser acelerado com o uso de uma sentinela. Para isso, atribui-se a chave de pesquisa ao registro contido na posição n+1 da lista. Este registro fictício funciona como uma sentinela. Mesmo no pior caso, a chave será encontrada na posição n+1.

  • Se o elemento alvo for encontrado em uma posição anterior a n+1, significa que o elemento está na lista.
  • Se o elemento alvo for encontrado somente na posição n+1, isso significa que o elemento não está presente na lista.

O objetivo do registro sentinela é usar o próprio elemento alvo como uma indicação de que não há mais registros a serem lidos, eliminando a necessidade de testar, a cada passo no laço, se já se chegou ao final da lista.

👨🏻‍💻 Algoritmo em C++ de pesquisa sequencial com sentinela

int sentinelLinearSearch(int chave, int vetor[], int tamVetor){
		int i = 0;
		vetor[n] = chave;
		
		while(v[i] != chave) i++;
		
		if (i < n ) return i;
		return -1;
}

🔎 Pesquisa binária

A pesquisa binária é um algoritmo eficiente. O processo básico consiste em comparar a chave de acesso (a chave pesquisada) com o registro central da lista. O comportamento depende do resultado dessa comparação:

  • Se a chave for menor, o registro alvo está na primeira metade da lista.
  • Se a chave for maior, o registro alvo está na segunda metade da lista.

Esse processo é repetido até que a chave seja encontrada. Para que a pesquisa binária funcione, é necessário que o vetor esteja ordenado.

👨🏻‍💻 Algoritmo em C++ de pesquisa binária

int binarySearch(int chave, int vetor[], int tamVetor){
		int inicio = 0, meio, fim = tamVetor-1;
		while(inicio <= fim){
				meio = (inicio + fim)/2;
				if(chave == vetor[meio]) return meio;
				else if (chave < vetor[meio]) fim = meio - 1;
				else inicio = meio + 1;
		}
		return -1;
}

👨🏻‍💻 Algoritmo em C++ de pesquisa binária recursiva

int binarySearchRecursive(int chave, int vetor[], int inicio, int fim){
		int meio = (inicio + fim)/2;
		if (inicio > fim) return -1;
		if (chave == vetor[meio]) return meio;
		else if (chave < vetor[meio]) return binarySearchRecursive(chave, vetor, inicio, meio-1);
		else return binarySearchRecursive(chave, vetor, meio+1, fim);
}

🔎 Pesquisa por Interpolação

A pesquisa por interpolação pode ser mais eficiente do que a pesquisa binária, especialmente quando as chaves estão uniformemente distribuídas dentro da lista. O algoritmo segue a mesma lógica da pesquisa binária, mas adota uma estratégia diferente para calcular o valor da variável "meio". Neste caso, o "meio" não será, necessariamente, o centro da lista.

👨🏻‍💻 Algoritmo em C++ de pesquisa por interpolação

int interpolationSearch(int chave, int vetor[], int tamVetor) {
		int ini = 0, meio, fim = tamVetor – 1;

		while (ini <= fim && chave >= vetor[ini] && chave <= vetor[fim]) {
				if (ini == fim) return v[ini] == chave ? ini : -1;	
				meio = ini + (((double)(fim - ini) * (chave – v[ini])) / (v[fim] - v[ini]));
				if (chave == v[meio]) return meio;
				else if (chave < v[meio]) fim = meio – 1;
				else inicio = meio + 1;
		}
	
	return -1;
}
