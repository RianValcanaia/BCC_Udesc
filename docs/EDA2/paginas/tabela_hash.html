Conceitos e Estratégia de Armazenamento

Problema de Busca

A maioria dos métodos de busca realiza comparações de chaves. Dada uma chave de pesquisa, comparam-se sucessivamente os dados armazenados.

  • Algoritmos eficientes requerem dados ordenados.
  • O limite inferior para algoritmos de ordenação é O(N log N).
  • Algoritmos de busca eficientes possuem complexidade O(log N).
  • A busca ideal seria com acesso direto (O(1)), sem varredura do conjunto de dados.

Acesso Direto em Vetores

Vetores usam índices para armazenar e acessar dados com complexidade O(1).

  • Contudo, não há mecanismo interno para calcular o índice a partir de um valor.
  • Assim, a busca pode se tornar O(N) no pior caso.
  • Uma alternativa são as Tabelas Hash, que calculam o índice com base em uma função de dispersão (hashing).

Tabelas Hash

Conceito

Também chamada de tabela de dispersão, é uma estrutura de dados que associa chaves a valores de forma eficiente, generalizando o conceito de vetor.

  • A posição de armazenamento é obtida por uma função de dispersão (hashing).
  • O objetivo é calcular um endereço único (índice) para cada chave.

Estrutura de uma tabela hash

typedef struct {
		int chave;
		void *valor;
}Entrada;

typedef struct {
		int total;
		int tamanho;
		Entrada **elementos;

} Hash;

Criar uma tabela hash

Hash *cria(int tamanho){
		Hash *hash = malloc(sizeof(Hash));
		hash->tamanho = tamanho;
		hash->total = 0;
		hash->elementos = malloc(tamanho *sizeof(Entrada*));
		for (int i = 0; i < tamanho; i++){
				hash->elementos[i] = NULL;
		}
		return hash;
}

Apagar/limpar a tabela hash

void liberar(Hash *hash) {
    if (hash != NULL) {
        for (int i = 0; i < hash->tamanho; i++) {
            if (hash->elementos[i] != NULL) {
                free(hash->elementos[i]);
            }
        }
        free(hash->elementos);
        free(hash);
    }
}

Funções de Dispersão (Hashing)

A função hashing transforma uma chave em uma posição na tabela.

Critérios desejáveis para uma boa função de dispersão:

  • Poucas colisões
  • Fácil de calcular
  • Distribuição uniforme

Exemplos de Funções Hash

1. Método da Divisão

Método da divisão: também conhecido como método da congruência linear, consiste em calcular a posição da chave a partir do resto da divisão.

  • H(c) = c % m
int hashingDivisao (int chave, int tamanho){
		return (chave & 0x7FFFFFFF) % tamanho;
}

2. Método da Multiplicação

Também conhecido como método da congruência linear multiplicativo, necessário determinar uma constante A, onde 0 < A < 1, use-se esta constante A para:

  • Multiplicar o valor da chave;
  • Parte fracionária resultate é multiplicada pelo tamanho da tabela.
int hashingMultiplicacao(int chave, int tamanho){
		float A = 0.61803387;  //constante 0 < A < 1
		float i = chave * A;  //realiza a multiplicacao
		i = i - (int)i;  //obtém a parte fracionária
		return (int)(tamanho * i);
}

3. Método da dobra

Utiliza um esquema de dobrar e somar os dígitos do valor para calcular a posição da chave na tabela, estratégia do método, considera uma chave como uma sequência de dígitos escritos em um papel, enquanto a chave for maior do que o tamanho da tabela, o papel é dobrado, os dígitos sobrepostos após a dobra são somados, desconsiderando dezena, forma mais fácil de implementar o método utiliza:

  • Operador ou exclusivo (xor)
  • Dobra é realizada de k em k bits
int hashingDobra(int chave, int tamanho){
		int k = 10; 
		int parte1 = chave >> k;
		int parte2 = chave & (tamanho-1);
		return (int)(parte1 ^ parte2);
}

Operações

Leitura e gravação

Quando não é necessário tratamento de colisão, utilzado em situações específicas quando o hashing é perfeito, neste cenário colisões não são aceitas, esforço computacional será O(1) sempre em todos os casos, quando conhecida a quantidade e quais dados serão armazenados, exemplo: tabela de símbolos de compiladores.

Insersão

Calcula a posição do elemento da tabela, a partir da função hashing, armazena o elemento na tabela na posição calculada.

void adicionar(Hash *hash, int chave, void *valor) {
    if (hash != NULL && hash->total < hash->tamanho) {
        int indice = hashingDivisao(chave, hash->tamanho);
        Entrada *entrada = malloc(sizeof(Entrada));
        entrada->chave = chave;
        entrada->valor = valor;
        hash->elementos[indice] = entrada;
        hash->total++;
    }
}

Busca

Calculada a posição do elemento na tabela a partir da função hashing, verifica a existência de um elemento na posição obtida, caso positivo retorna o elemento encontrado, caso contrário nulo.

void *buscar(Hash *hash, int chave) {
    if (hash != NULL) {
        int indice = hashingDivisao(chave, hash->tamanho);
        if (hash->elementos[indice] != NULL) {
            return hash->elementos[indice]->valor;
        }
    }
    return NULL;
}

Tratamento de colisão

Endereçamento Aberto

Armazena o valor no próximo espaço livre na tabela (rehash).

Vantagens

  • Busca mais rápida (própria tabela)
  • Não usa espaço adicional

Desvantagens

  • Complexidade pode chegar a O(N) na inserção
  • Sofre com clusters de colisão

Endereçamento Separado (Encadeamento)

Cada posição da tabela é uma lista encadeada.

Estrutura

typedef struct {
		int chave;
		void *valor; 
} Entrada;

typedef struct _no{
		Entrada *entrada;
		struct _no *próximo;
} No; 

typedef struct{
		int total;
		int tamanho;
		No **elementos;
} Hash;

Criar

Hash *criar(int tamanho) {
    Hash *hash = malloc(sizeof(Hash));
    hash->tamanho = tamanho;
    hash->total = 0;
    hash->elementos = malloc(tamanho * sizeof(No*));
    for (int i = 0; i < tamanho; i++) {
        hash->elementos[i] = NULL;
    }
    return hash;
}

Inserção com colisão

void adicionar(Hash *hash, int chave, void *valor) {
    if (hash != NULL) {
        int indice = hashingDivisao(chave, hash->tamanho);
        Entrada *entrada = malloc(sizeof(Entrada));
        entrada->chave = chave;
        entrada->valor = valor;

        No *no = malloc(sizeof(No));
        no->entrada = entrada;
        no->proximo = hash->elementos[indice];
        hash->elementos[indice] = no;
        hash->total++;
    }
}

Buscar 

void *buscar(Hash *hash, int chave) {
    if (hash != NULL) {
        int indice = hashingDivisao(chave, hash->tamanho);
        No *no = hash->elementos[indice];
        while (no != NULL) {
            if (no->entrada->chave == chave) {
                return no->entrada->valor;
            }
            no = no->proximo;
        }
    }
    return NULL;
}

Vantagens das Tabelas Hash

  • Busca com tempo médio O(1)
  • Simples de implementar
  • Generalização de vetores

Desvantagens

  • Alto custo para ordenar os dados
  • Necessidade de conhecer o número de elementos a priori
  • Pior caso: complexidade O(N) devido a colisões

Aplicações

  • Cache de dados (bancos em memória)
  • Criptografia (MD5, SHA, armazenamento de senhas)
  • Compiladores (tabela de símbolos)
