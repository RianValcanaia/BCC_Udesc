Introdução às Árvores

As estruturas de dados tradicionais, como as listas, representam dados de forma sequencial. Já as árvores são adequadas para representar relações hierárquicas. Elas possuem uma definição recursiva: cada nó de uma árvore forma uma subárvore.

Uma árvore é composta por:

  • Um conjunto de nós;
  • Um nó especial chamado raiz, que está no topo da hierarquia;
  • Cada nó pode ter múltiplos filhos;
  • Nós que não possuem filhos são chamados de folhas;
  • Os nós representam os vértices da árvore;
  • As conexões entre os nós são chamadas de arestas;
  • As árvores são organizadas em níveis.

Conceitos Básicos

  • Cada nó (exceto a raiz) possui exatamente um pai.
  • Cada nó possui uma coleção de filhos, que pode ser vazia.
  • Em árvores binárias, cada nó pode ter no máximo dois filhos.
  • O nó raiz não possui pai.
  • Nós folhas são aqueles que não possuem filhos.
  • O grau de um nó representa a quantidade de filhos que ele possui.
  • O grau de saída de um nó é o número de filhos que ele tem.
  • O grau da árvore é o maior grau entre todos os nós.
  • Uma floresta é uma coleção de árvores disjuntas.

Tipos de Árvores

  • Árvore cheia de grau D: todos os nós (exceto os nós folha) possuem exatamente D filhos.
  • Árvore binária cheia (grau 2):
    • Todos os nós (exceto os nós folha) possuem exatamente dois filhos;
    • Também chamadas de árvores binárias perfeitamente balanceadas;
    • Podem ser implementadas sequencialmente usando vetores.

Representação de Expressões Aritméticas

Em uma árvore binária usada para representar expressões:

  • Os nós folhas representam os operandos;
  • Os nós internos representam os operadores.

Caminho

  • Um caminho é uma sequência de nós distintos conectados pela relação de pai e filho.
  • Existe apenas um caminho da raiz até qualquer outro nó da árvore.
  • O comprimento de um caminho refere-se ao número de nós que ele contém.

Profundidade

  • A profundidade de uma árvore é o comprimento do maior caminho da raiz até a folha mais distante.
  • Também pode ser chamada de altura da árvore.

Largura

  • A largura de uma árvore é o maior número de nós encontrados em um mesmo nível.

Representação de árvores em C

Estrutura de Nó e de Árvore

typedef struct no {
    struct no* pai;
    struct no* esquerda;
    struct no* direita;
    float registro;
} No;

typedef struct arvore {
    struct no* raiz;
} Arvore;

Operações básicas

Criar uma árvore

Arvore* cria() {
    Arvore* arvore = malloc(sizeof(Arvore));
    arvore->raiz = NULL;
    return arvore;
}

Cria um nó

No* criarNo(float valor, No* pai) {
    No* novo = (No*)malloc(sizeof(No));
    if (novo != NULL) {
        novo->registro = valor;
        novo->esquerda = NULL;
        novo->direita = NULL;
        novo->pai = pai;
    }
    return novo;
}

Inserir um novo elemento na árvore

No* adiciona(Arvore* arvore, No* pai, float valor) {
    No* no = malloc(sizeof(No));
    no->pai = pai;
    no->esquerda = NULL;
    no->direita = NULL;
    no->registro = valor;

    if (pai == NULL) {
        arvore->raiz = no;
    }

    return no;
}

Remover o elemento da árvore

void remove(Arvore* arvore, No* no) {
    if (no->esquerda != NULL) remove(arvore, no->esquerda);
    if (no->direita != NULL) remove(arvore, no->direita);

    if (no->pai == NULL) {
        arvore->raiz = NULL;
    } else {
        if (no->pai->esquerda == no)
            no->pai->esquerda = NULL;
        else
            no->pai->direita = NULL;
    }

    free(no);
}

Percorrer a árvore (pré-ordem)

void percorrer(No* no) {
    if (no != NULL) {
        printf("%f\n", no->registro);
        percorrer(no->esquerda);
        percorrer(no->direita);
    }
}

Percorrer a árvore (em-ordem)

void percorrer(No* no) {
    if (no != NULL) {
        percorrer(no->esquerda);
        printf("%f\n", no->registro);
        percorrer(no->direita);
    }
}

Percorrer a árvore (pós-ordem)

void percorrer(No* no) {
    if (no != NULL) {
        percorrer(no->esquerda);
        percorrer(no->direita);
        printf("%f\n", no->registro);
    }
}

Verificar se a árvore está vazia

int is_empty(Arvore* arv) {
    return (arv->raiz == NULL);
}

Liberar memória da árvore

void liberaNos(No* no) {
    if (no == NULL) return;
    liberaNos(no->esquerda);
    liberaNos(no->direita);
    free(no);
}

void liberaArvore(Arvore* arv) {
    if (arv == NULL) return;
    liberaNos(arv->raiz);
    free(arv);
}

Busca em uma Árvore Binária

A busca em uma árvore binária consiste em visitar seus nós segundo uma determinada estratégia de percurso. O tipo de busca determina a ordem em que os nós são visitados e pode ser usada para imprimir, processar ou buscar valores.

Busca em Profundidade (DFS – Depth-First Search)

Na busca em profundidade, a exploração da árvore segue o caminho dos ramos até alcançar o nó folha antes de retroceder. O processo é recursivo, explorando cada subárvore a partir da esquerda.

As principais variações da DFS são:

1. Pré-Ordem (raiz → esquerda → direita)

Visita a raiz antes de seus filhos. Útil para copiar uma árvore.

void preOrdem(No* no) {
    if (no == NULL) return;

    printf("%.1f ", no->registro);       // raiz
    preOrdem(no->esquerda);              // esquerda
    preOrdem(no->direita);               // direita
}

2. Em-Ordem (esquerda → raiz → direita)

Visita os nós em ordem crescente (caso a árvore seja de busca). Muito usada para listar os dados em ordem.

void emOrdem(No* no) {
    if (no == NULL) return;

    emOrdem(no->esquerda);               // esquerda
    printf("%.1f ", no->registro);       // raiz
    emOrdem(no->direita);                // direita
}

3. Pós-Ordem (esquerda → direita → raiz)

Visita os filhos antes do pai. É útil para liberar memória ou destruir a árvore.

void posOrdem(No* no) {
    if (no == NULL) return;

    posOrdem(no->esquerda);              // esquerda
    posOrdem(no->direita);               // direita
    printf("%.1f ", no->registro);       // raiz
}

Busca em Largura (BFS – Breadth-First Search)

A busca em largura percorre a árvore nível por nível, da esquerda para a direita. Ou seja, primeiro são visitados todos os nós do nível 0 (raiz), depois os do nível 1, depois os do nível 2, e assim por diante.

Essa estratégia exige que os nós sejam processados na ordem em que foram descobertos, e para isso utiliza-se uma fila.

Estrutura da fila

typedef struct fila {
    No* no;
    struct fila* prox;
} Fila;

Enfileirar

void enfileirar(Fila** inicio, Fila** fim, No* no) {
    Fila* novo = malloc(sizeof(Fila));
    novo->no = no;
    novo->prox = NULL;

    if (*fim != NULL)
        (*fim)->prox = novo;
    else
        *inicio = novo;

    *fim = novo;
}

Desenfileirar

No* desenfileirar(Fila** inicio, Fila** fim) {
    if (*inicio == NULL) return NULL;

    Fila* temp = *inicio;
    No* no = temp->no;
    *inicio = temp->prox;

    if (*inicio == NULL)
        *fim = NULL;

    free(temp);
    return no;
}

Algoritmo de Busca em Largura (BFS)

void bfs(No* raiz) {
    if (raiz == NULL) return;

    Fila* inicio = NULL;
    Fila* fim = NULL;

    enfileirar(&inicio, &fim, raiz);

    while (inicio != NULL) {
        No* atual = desenfileirar(&inicio, &fim);
        printf("Visitando: %.1f\n", atual->registro);

        if (atual->esquerda != NULL)
            enfileirar(&inicio, &fim, atual->esquerda);

        if (atual->direita != NULL)
            enfileirar(&inicio, &fim, atual->direita);
    }
}

Árvore Binária de Busca (ABB)

Uma Árvore Binária de Busca (ABB) é uma estrutura de dados hierárquica utilizada para armazenar informações de forma ordenada, permitindo operações eficientes de busca, inserção e remoção.

Cada nó da árvore contém:

  • Uma chave de busca (como um número de identidade, CPF, etc.);
  • Ponteiros para os filhos à esquerda e à direita;
  • (Opcionalmente) outros dados associados à chave.

Propriedades de uma ABB

Uma árvore binária de busca obedece às seguintes regras:

  • Todo item na subárvore à esquerda de um nó é menor ou igual à sua chave.
  • Todo item na subárvore à direita é maior do que a chave.
  • Cada subárvore também deve ser uma ABB.
Com essas regras, é possível localizar qualquer elemento em O(log n) no melhor caso (com a árvore balanceada).

Inserção em uma ABB

A inserção utiliza uma abordagem similar ao percurso pré-ordem:

  1. Se a árvore está vazia, o novo valor é colocado na raiz.
  1. Caso contrário, percorre-se a árvore comparando o valor a ser inserido com os nós atuais:
    • Se for menor ou igual, vai para a subárvore esquerda.
    • Se for maior, vai para a subárvore direita.
  1. O ponteiro do pai é atualizado adequadamente.
void insere(Arvore *arv, float valor) {
    No *atual = arv->raiz, *pai = NULL;

    while (atual != NULL) {
        pai = atual;
        if (valor <= atual->registro)
            atual = atual->esquerda;
        else
            atual = atual->direita;
    }

    No *novo = criarNo(valor, pai);

    if (pai == NULL)
        arv->raiz = novo;
    else if (valor < pai->registro)
        pai->esquerda = novo;
    else
        pai->direita = novo;
}

Busca em uma ABB

A busca também é feita de forma recursiva ou iterativa, comparando o valor com os nós da árvore:

  1. Compare o valor com a raiz.
  1. Se for menor, vá para a subárvore esquerda.
  1. Se for maior, vá para a subárvore direita.
  1. Repita até encontrar o valor ou atingir um ponteiro NULL.
No* busca(Arvore *arv, float valor) {
    No *atual = arv->raiz;

    while (atual != NULL) {
        if (valor == atual->registro)
            return atual;
        else if (valor < atual->registro)
            atual = atual->esquerda;
        else
            atual = atual->direita;
    }

    return NULL;
}

Remoção em Árvores Binárias de Busca (ABB)

A remoção em uma ABB é mais complexa do que as operações de inserção e busca. O processo deve preservar a estrutura ordenada da árvore após a remoção do nó.

Visão Geral

Existem três casos principais a serem considerados ao remover um nó:

  1. Nó folha (sem filhos): basta removê-lo.
  1. Nó com um único filho: o filho substitui o nó removido.
  1. Nó com dois filhos: substituímos o nó pelo maior valor da subárvore esquerda (ou o menor valor da subárvore direita).

Estratégia

  • O item máximo da árvore pode ser encontrado indo sempre para o filho à direita até encontrar o último nó.
  • Se esse nó tiver um filho à esquerda, ele precisa ser ligado ao nó anterior (pai).
  • O nó máximo encontrado será usado como substituto quando o nó removido tiver dois filhos.

Função para buscar e remover o maior da subárvore esquerda

No* remmax(No* no) // busca o antecessor em ordem
		No* atual = no->esquerda;
    No* pai = no;

    while (atual->direita != NULL) {
        pai = atual;
        atual = atual->direita;
    }

    // Remove 'atual' da subárvore
    if (pai == no) {
        // O maior é o filho esquerdo direto
        no->esquerda = atual->esquerda;
    } else {
        pai->direita = atual->esquerda;
    }

    atual->esquerda = NULL;
    atual->direita = NULL;

    return atual;
}

Função principal de remoção

void remover(Arvore* arvore, float valor) {
    No* no = buscar(arvore, valor);
    if (no == NULL) return;

    No* pai = no->pai;
    int filho_esq = (pai != NULL && pai->esquerda == no);

    // Caso 1: Nó folha
    if (no->esquerda == NULL && no->direita == NULL) {
        if (pai == NULL) arvore->raiz = NULL;
        else if (filho_esq) pai->esquerda = NULL;
        else pai->direita = NULL;
    }

    // Caso 2: Apenas filho à direita
    else if (no->esquerda == NULL) {
        if (pai == NULL) arvore->raiz = no->direita;
        else if (filho_esq) pai->esquerda = no->direita;
        else pai->direita = no->direita;

        no->direita->pai = pai;
    }

    // Caso 3: Apenas filho à esquerda
    else if (no->direita == NULL) {
        if (pai == NULL) arvore->raiz = no->esquerda;
        else if (filho_esq) pai->esquerda = no->esquerda;
        else pai->direita = no->esquerda;

        no->esquerda->pai = pai;
    }

    // Caso 4: Dois filhos
    else {
        No* maximo = remmax(no); // maior valor da subárvore esquerda

        maximo->esquerda = no->esquerda;
        if (maximo->esquerda != NULL)
            maximo->esquerda->pai = maximo;

        if (no->direita != maximo) {
            maximo->direita = no->direita;
            if (no->direita != NULL)
                no->direita->pai = maximo;
        }

        maximo->pai = pai;

        if (pai == NULL)
            arvore->raiz = maximo;
        else if (filho_esq)
            pai->esquerda = maximo;
        else
            pai->direita = maximo;
    }

    free(no);
}

Árvore de Huffman

A árvore de Huffman é uma árvore binária completa usada para compressão de dados, com o objetivo de reduzir o espaço de armazenamento ou o tempo de transmissão. Ela representa um código de tamanho variável, minimizando o número de bits necessários para representar os dados, diferentemente do código ASCII, que tem tamanho fixo.

Propriedades:

  • É uma árvore binária estritamente completa.
  • É construída unindo recursivamente os dois símbolos de menor frequência, criando nós intermediários com a soma de suas frequências.
  • O código gerado é livre de prefixo, ou seja, o código de um caractere nunca é prefixo do código de outro.
  • É dependente do conteúdo: o código de Huffman é específico para cada arquivo.

Construção da Árvore de Huffman

  1. Para cada caractere c do texto s, calcule sua frequência f(c).
  1. Crie uma folha para cada caractere com f(c) > 0.
  1. Construa uma floresta com essas folhas.
  1. Enquanto houver mais de uma árvore na floresta:
    • Remova as duas árvores de menor frequência (Ae e Ad);
    • Crie uma nova árvore A com frequência f = f(Ae) + f(Ad);
    • Insira A de volta na floresta.
  1. A árvore final é a árvore de Huffman.
  1. Atribua 0 para arestas à esquerda e 1 para à direita. O código de cada caractere é o caminho da raiz até sua folha.

Estrutura de uma árvore de Huffman

typedef struct arvoreHuffman {
		char chr;
		int freq;
		struct arvoreHuffman *esq, *dir;
} *Arvh;

Calculo das frequências

Veremos uma implementação do código de Huffman. Dada
uma cadeia de caracteres s, representando o conteúdo de um
arquivo a ser comprimido, é criado um vetor de inteiros f tal
que, para cada caractere ASCII c, f[c] é a frequência de c em s:

int* freq(char *s) {
    static int f[256];
    for (int i = 0; i < 256; i++) f[i] = 0;
    for (int i = 0; s[i]; i++) f[(unsigned char)s[i]]++;
    return f;
}

Criação da árvore de Huffman

Para criar uma árvore de Huffman, uma função recebe como
entrada um caractere, sua frequência e duas árvores de
Huffman e, como saída, ela devolve o endereço de um nó
criado e preenchido com esses valores.
Assim, cada folha numa árvore de Huffman guardará um
caractere e sua frequência.

Arvh criaAH(char chr, int freq, Arvh esq, Arvh dir) {
    Arvh nova = malloc(sizeof(struct arvoreHuffman));
    nova->chr = chr;
    nova->freq = freq;
    nova->esq = esq;
    nova->dir = dir;
    return nova;
}

Remoção de uma árvore com menor frequência

Para a remocao de uma árvore da floresta sabendo que F é um vetor de árvores em ordem decrescente de peso, basta remover a ultima árvore do vetor retornando ela como resposta:

Arvh remf(Arvh F[], int *m) {
    if (*m == 0) return NULL;
    return F[--(*m)];
}

Inserção Ordenada (decrescente por frequência)

Uma floresta será representada por um vetor F com m
árvores de Huffman, em ordem decrescente de peso. Para
garantir a ordenação da floresta, usaremos a função abaixo.
Dadas uma árvore A e uma floresta F com m árvores, essa
função insere a árvore A em F, em ordem decrescente de
peso, e incrementa m. O parâmetro m, que indica o tamanho
da floresta, é passado por referência.

void inst(Arvh A, Arvh F[], int *m) {
    int i = *m;
    while (i > 0 && F[i - 1]->freq < A->freq) {
        F[i] = F[i - 1];
        i--;
    }
    F[i] = A;
    (*m)++;
}

Construção da árvore de Huffman

Arvh huffman(char *s) {
    Arvh F[256];
    int m = 0;
    int *f = freq(s);

    for (int c = 0; c < 256; c++) {
        if (f[c] > 0)
            inst(criaAH((char)c, f[c], NULL, NULL), F, &m);
    }

    while (m > 1) {
        Arvh d = remf(F, &m);
        Arvh e = remf(F, &m);
        inst(criaAH('-', d->freq + e->freq, d, e), F, &m);
    }

    return F[0];
}

Geração dos Códigos de Huffman

Construida a árvore A, agora podemos gerar o código binário a partir dela e da própria cadeia de caracteres s. Para definir qual a sequência binária para cada letra do alfabeto:

void tabela(Arvh A, char *H[]) {
    static char c[256];
    static int t = -1;

    if (A == NULL) return;

    if (A->esq == NULL && A->dir == NULL) {
        H[(unsigned char)A->chr] = strndup(c, t + 1);
    } else {
        t++;
        c[t] = '0'; tabela(A->esq, H);
        c[t] = '1'; tabela(A->dir, H);
        t--;
    }
}

Sabendo agora o código de cada letra, podemos agora gerar o código a partir da string e da árvore de Huffman:

void comprimir(char *s, Arvh A) {
    char *T[256];
    for (int c = 0; c < 256; c++) T[c] = NULL;

    tabela(A, T);

    for (int i = 0; s[i]; i++)
        printf("%s", T[(unsigned char)s[i]]);

    for (int c = 0; c < 256; c++) free(T[c]);
}

Decodificando/descomprimindo

Para descomprimir uma cadeia de dígitos d, basta ter a árvore de Huffman A, que foi usada para comprimi-la. A cada valor binário, a árvore é percorrida até chegar no nos nós folha:

void descomprimir(char *d, Arvh A) {
    if (A == NULL) return;

    Arvh n = A;
    for (int i = 0; d[i]; i++) {
        n = (d[i] == '0') ? n->esq : n->dir;

        if (n->esq == NULL && n->dir == NULL) {
            printf("%c", n->chr);
            n = A;
        }
    }
}
